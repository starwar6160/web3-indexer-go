# Concurrency Overview

## Current Concurrency Primitives

The **web3-indexer-go** project primarily uses the following concurrency mechanisms:

- **Mutexes (`sync.Mutex` / `sync.RWMutex`)** – Guard access to shared maps and in‑memory caches, especially in the `engine` package (e.g., `address_manager.go`, `reality_collapse.go`).
- **Channels** – Used for worker pools and streaming blocks from the RPC source to the processing pipeline.
- **Atomic operations (`sync/atomic`)** – Lightweight counters for metrics and request IDs.
- **WaitGroups (`sync.WaitGroup`)** – Coordinate shutdown of background goroutines.
