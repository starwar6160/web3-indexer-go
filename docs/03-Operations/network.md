# 网络访问与安全策略 (80分安全架构)

本项目在关闭公网 DMZ 后，采用基于 **IP Binding** 和 **Tailscale** 的分级访问控制策略。

## 1. 访问控制列表 (ACL)

| 服务 | 端口 | 监听范围 | 访问权限 |
| --- | --- | --- | --- |
| **API / Dashboard** | 8080 / 8081 | `0.0.0.0` | 局域网/Tailscale 可访问 |
| **Grafana** | 4000 | `0.0.0.0` | 局域网/Tailscale 可访问 |
| **PostgreSQL** | 15432 / 15433 | `127.0.0.1`, `100.66.253.77` | **仅限宿主机与 Tailscale 节点** |

## 2. 安全背景
- **防御目标**: 屏蔽公网自动化脚本的数据库盲扫。
- **管理手段**: 通过 Tailscale 建立私有管理平面，数据库不暴露于任何非私有接口。

## 3. 运维验证指令
检查宿主机真实监听状态：
```bash
sudo ss -tunlp | grep -E '15432|15433'
```
预期应只看到 `127.0.0.1` 和 `100.66.253.77` 的条目。
